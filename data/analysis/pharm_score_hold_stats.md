# 팜이데일리 API `score` × 홀딩 수익률 백테스트 요약

원천 데이터는 `data/analysis/pharm_score_hold_stats.json`과 동기화되어 있습니다. 갱신하려면:

```bash
python3 scripts/pharm_score_hold_stats.py
```

## 정의

| 항목 | 내용 |
|------|------|
| 표본 | `data/somedaynews_article_tickers.json` + EODHD 매니페스트 (**intraday_ok** & **eod_ok**) |
| `score` | 팜이데일리 API 노출 점수(정수). **클수록 좋음**으로 해석. 없으면 `score_null` |
| 홀딩 | **거래일** 기준 1, 2, 3, 5일 (진입일 EOD 인덱스 + h) |
| **진입 F** | `published_at` 이후 첫 장중 5분봉 종가 (`analyze_10m_return_path` / `entry_hold_analysis` 와 동일) |
| **진입 A** | 기사 KST 달력일의 첫 매칭 거래일(T0) 장중 **마지막** 5분봉 종가 |
| 청산 | 해당 거래일 장중 마지막 5분봉 종가 |
| 수익률 | (청산 − 진입) / 진입 |

## 표본 크기

| 지표 | 값 |
|------|-----|
| 매니페스트 통과 이벤트 수 | 1918 |
| 진입 F가 최소 1개 홀딩에서 성공한 이벤트 수 | 1448 |
| 진입 A가 최소 1개 홀딩에서 성공한 이벤트 수 | 1351 |

> 유의성은 보통 p &lt; 0.05를 참고합니다. 여러 홀딩·여러 검정을 동시에 보므로 **다중 비교**를 고려해 보수적으로 해석하는 것이 좋습니다.

## 해석 메모

1. **Spearman**(점수 순위 vs 수익률)과 **Mann–Whitney**(고점수 ≥4 vs 중간 2–3)는 대부분 **유의하지 않음**(p &gt; 0.05).
2. **진입 F, H=1**에서만 Kruskal–Wallis(점수 2·3·4·5)가 **p ≈ 0.026**으로 0.05 미만. 다만 **여러 검정·여러 홀딩**을 함께 보면 우연에 가까울 수 있어, **단독 강한 신호**로 보기는 어렵습니다.
3. 서술적으로는 **장기(H=5)·진입 F**에서 `score_null`의 평균·승률이 다른 버킷보다 불리한 편이 자주 나타나지만, 이는 **별도 검정으로 분리해 두지 않았음**을 참고하세요.
4. `score_1` 버킷은 표본이 작아 **해석에 제한**이 있습니다.
