# Look-ahead 없는 F 진입 알고리즘 백테스트

**당일 전체 고저(range) 같은 사후 정보는 쓰지 않고**, 전일 EOD·당일 시가·F 확정가 등으로 필터·점수를 만든 뒤 승률 60%+ 구간을 탐색한 결과. (F·앵커 항목은 **봉이 끝난 뒤에만** 아는 값이므로, 실전에서는 조건부 진입으로 해석할 것.)

- **스크립트**: `scripts/algo_no_lookahead_backtest.py`
- **필터 탐색**: `scripts/find_60pct_no_lookahead.py`
- **결과 JSON**: `data/analysis/algo_no_lookahead_backtest.json`
- **재실행**: `python3 scripts/algo_no_lookahead_backtest.py`

---

## 사용한 피처 (전부 진입 이전에 확정)

| 피처 | 뜻 | 언제 확정? |
|------|-----|-----------|
| `pub_hour` | 기사 공개 시각 (KST) | 기사 공개 순간 |
| `close_vs_ma20` | 전일 종가 / 20일 이평 − 1 | 전일 장 마감 |
| `gap_open_pct` | 앵커가 잡힌 세션 시가 / **직전 거래일** 종가 − 1 | 해당 세션 시가 확정 시 |
| `ret_1d_pre` | 전전일 종가 → 전일 종가 수익률 | 전일 장 마감 |
| `entry_vs_prev_close` | F진입가 / 전일 종가 − 1 | F봉 확정 시 |
| `entry_vs_prev_low` | F진입가 / 전일 저가 − 1 | F봉 확정 시 |

> **이전 버전(algo_dip_entry_backtest)과 차이**: `entry_in_range`(당일 고가·저가 레인지 대비 위치)는 **장 끝나야 확정**되므로 이번에는 사용하지 않았습니다.

## 전략 정의

| 이름 | 조건 | 쉬운 말 |
|------|------|---------|
| **baseline** | 없음 | F 진입 전체 |
| **S1** | `pub_hour < 8` | 장 전(오전 8시 이전) 기사만 |
| **S2** | `close_vs_ma20 < 0` & `pub_hour < 8` | 전일 종가가 20일 이평 밑 + 장 전 기사 |
| **S3** | `close_vs_ma20 < −7%` & `pub_hour < 8` | 전일 종가가 이평 크게 밑 + 장 전 기사 |
| **S4** | `close_vs_ma20 < 0` & `gap_open_pct < −2%` | 이평 밑 + 당일 시가 갭 하락 |
| **S5** | `pub_hour < 8` & `ret_1d_pre < −2%` | 장 전 기사 + 전일 하락 |
| **S6** | `entry_vs_prev_low < −3%` | F진입가가 전일 저가보다 3% 이상 아래 |
| **S7** | `entry_vs_prev_close < −7%` & `entry_vs_prev_low < −3%` | 진입가가 전일 대비 크게 하락 |

## 해석

### S4가 왜 좋은가 (쉬운 말)

**"어제 종가가 20일 이평보다 낮고, 오늘 시가가 어제보다 2% 이상 갭 하락해서 시작한 날"**에 뉴스가 나오면 매수.
→ 이미 **눌려 있는 종목**이 **시가에서 한 번 더 빠져서 열렸을 때** 뉴스가 걸린 경우.
→ 평균 반등이 강하다.

### 주의사항

1. **n=30 전후**로 표본이 적습니다. 통계적 신뢰도가 높다고 보기 어렵고, **과최적화 가능성**이 있습니다.
2. 그래도 **진입 시점에 모르는 정보는 전혀 쓰지 않았다**는 점에서 이전 `entry_in_range` 기반 전략보다 **실전 적용 가능성이 높습니다**.
3. **S1(장 전 기사, n=422, 승률 58.3%)이 표본이 가장 많고 안정적**입니다. 단순하면서도 baseline(48.2%) 대비 10%p 이상 높습니다.

## 실전 적용 순서 제안

1. **가장 단순**: S1 — 오전 8시 이전에 나온 유료 기사 종목 → F 매수 → 당일 장종 청산 (58.3%)
2. **승률 높지만 기회 적음**: S4 — 이평 밑 + 갭 하락 날 → F 매수 (86.7%, n=30)
3. **절충**: S2 — 이평 밑 + 장 전 기사 (76.3%, n=76)

## 기사 점수화 (이벤트별 0~100)

동일 피처로 **기사×종목 1건**마다 점수를 붙이려면:

- **스크립트**: `scripts/article_news_score.py`
- **결과**: `data/analysis/article_news_scores.json` (`rows`: 전체, 정렬은 점수 내림차순)
- **실행**: `python3 scripts/article_news_score.py` · 샘플만: `--limit 50` · 파일 없이 콘솔만: `--no-json`
- **UI**: 기사 상세 `/article/[newsId]` 퀀트 사이드바에서 총점(참고) 카드 **바로 아래**에 **「알고리즘 시그널」`xx.x/100`**과 **가로 게이지 막대**(총점 행과 동일 패턴)로 표시. 데이터는 `GET /api/quant/insight` 응답의 `news_signal` 필드 (`lib/news-signal-score.ts`가 `article_news_scores.json` 조회).

**합산(100점 만점)**

| 축 | 만점 | 요지 |
|----|------|------|
| 시간대 | 15 | KST 8시 전이면 높음 |
| MA20 이격 | 20 | 전일이 이평 아래로 많이 눌릴수록 높음 |
| 전일 수익 | 15 | 전일 하락 클수록 높음 |
| 갭 | 15 | 시가 갭 하락 클수록 높음 |
| 앵커 위치 | 25 | F가 전일 종가·저가 대비 아래로 깊을수록 높음 |
| 보너스 | 10 | S2·S4·S5와 비슷한 조합이면 가산 |

각 행에 `flags`(S1~S7 해당 여부)와 `breakdown`이 들어 있음. 가중치는 백테스트 탐색 결과에 맞춘 **휴리스틱**이며, 운용 전 워크포워드로 다시 맞출 수 있음.
