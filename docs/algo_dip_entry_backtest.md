# 저점 진입(Dip-Entry) 필터 백테스트

유료 기사 **F 진입**(공개 시각 직후 첫 장중 5분봉 종가)에 대해, 당일 가격 레인지 대비 **저점 근처**에서만 매수하는 필터와 그 변형을 백테스트한 결과를 정리합니다.

- **스크립트**: `scripts/algo_dip_entry_backtest.py`
- **결과 JSON**: `data/analysis/algo_dip_entry_backtest.json` (`generated_at` 기준 스냅샷: 2026-04-17)
- **재실행**: `python3 scripts/algo_dip_entry_backtest.py`

---

## 방법론

| 항목 | 정의 |
|------|------|
| **대상** | `somedaynews_article_tickers.json` + manifest, intraday·EOD 모두 OK, `excluded_tickers` 제외 |
| **진입** | F — `published_at` 이후 종료되는 첫 장중 5분봉 **종가** |
| **청산** | 진입 세션 기준 **N거래일** 뒤 해당일 **장종 5분봉 종가** (EOD가 짧으면 `min(진입+N, 마지막 거래일)` 클립) |
| **승률** | 청산 수익률 > 0 비율 |

### 피처 정의

- **`entry_in_range`** = (진입가 − 당일 저가) / (당일 고가 − 당일 저가)  
  - 당일 장중 5분봉만으로 고가·저가 계산.
- **`ret_1d_pre`** = 이벤트일 **직전 거래일** 수익률 (T−2 종가 → T−1 종가, EOD `close`).

---

## 주의: 생존·사후 편향

1. **`entry_in_range`는 당일 고가·저가를 알아야 계산**할 수 있습니다. 장중에 “지금 진입이 하단 20%인가?”를 확정하려면 **그 시점까지의 누적 고저**로 바꾸거나, **장 후 스크리닝 → 익일 매수** 등 규칙을 따로 두어야 합니다.
2. **`ret_1d_pre`는 진입 전날까지의 정보**로, 당일 장중 진입과 함께 쓰는 것은 일반적으로 문제 없지만, 전략 C·D는 **표본 수(n)가 작아집니다**.
3. 베이스라인 대비 승률·평균이 좋아 보여도 **과최적화·다중검정** 가능성이 있습니다.

## 전략 정의

| 코드 | 조건 |
|------|------|
| **baseline** | 필터 없음 (F 진입 전체) |
| **A** | `entry_in_range` < 0.20 |
| **B** | `entry_in_range` < 0.10 |
| **C** | A + `ret_1d_pre` < −1% |
| **D** | A + `ret_1d_pre` < −3% |
| **E** | `entry_in_range` < 0.05 |
