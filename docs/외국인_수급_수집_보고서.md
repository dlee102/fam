## 1. 개요

본 보고서는 KOSCOM(Korea Securities Computer Corp.) 정보분배 로그 원본을 분석하여 **외국인 수급(투자자별 매매) 데이터**가 어디에 있으며, 어떻게 수집·파싱할 수 있는지 정리한다.

### 1.1 로그 파일 구조

- **형식**: 바이너리 스트림, 레코드 구분자 `0xFF` (1바이트)
- **파일명**: `{포트번호}_{타임스탬프}.log` (예: `23103_1735743918.log`)
- **인코딩**: ASCII 호환 (숫자·영문)

### 1.2 포트별 채널 구성 (KOSCOM 명세)

| 포트 | 정보상품 | 용도 | 외국인 수급 |
|------|----------|------|-------------|
| 22102 | 증권A 실시간 | 체결(A301S) | ❌ |
| 22103 | 증권A 배치/지수 | C001S, C301S 등 | ✅ C001S |
| 22104 | 증권A 실시간 | 우선호가(B601S) | ❌ |
| 23102 | 증권B 실시간 | 체결(A301Q) | ❌ |
| **23103** | **증권B 배치/지수** | **C001Q, C001X** | **✅ C001Q** |
| **23104** | 증권B 실시간 | 우선호가(B601Q) | ❌ |
| 29102, 29103 | 지수상품 | IA000, IB000 | ❌ |

## 2. 외국인 수급 데이터가 있는 전문

### 2.1 업종별 투자자별 통계 (C001) — **로그에 존재**

| TR코드 | 시장 | 설명 | 제공 주기 |
|--------|------|------|-----------|
| C001S | 코스피(STK) | 업종별 투자자별 통계 | 09:00~18:00, 90초 주기 |
| **C001Q** | **코스닥(KSQ)** | **업종별 투자자별 통계** | **90초 주기** |
| C001X | 코넥스(KNX) | 업종별 투자자별 통계 | 90초 주기 |

**투자자 구분 코드 (TR코드 마지막 2자리)**:

| 코드 | 투자자 | 23103 로그 건수 |
|------|--------|-----------------|
| C001Q**100** | 개인 | 2,232건 |
| C001Q**101** | 기관 | 2,604건 |
| C001Q**102** | **외국인** | **2,232건** |
| C001Q**103** | 기타 | 372건 |

### 2.2 종목별 투자자별 종가통계 (C101) — **현재 로그에 없음**

| TR코드 | 시장 | 설명 | 제공 시점 |
|--------|------|------|-----------|
| C101S | 코스피 | 종목별 투자자별 종가통계 | 15:30, 18:00 (배치) |
| C101Q | 코스닥 | 종목별 투자자별 종가통계 | 15:30, 18:00 |
| C101X | 코넥스 | 종목별 투자자별 종가통계 | 15:30, 18:00 |

- **종목 단위** 외국인 순매수/순매도 제공
- 인터넷분배 상품에는 증권 종목별 투자자별 통계가 **없음** (명세 참조)
- 전용선·배치 채널에서만 수신 가능

### 3.2 C001Q102 (외국인) 레코드 구조

```
[TR+투자자 9바이트][업종코드 등 91바이트][타임스탬프]
C001Q102    + 001000QGG01PKRD020040006000001535891000000028408644955.000000001294846000000014662177039.00  + 1735779690028
```

- **총 길이**: 약 100~117바이트 (0xFF 제외)
- **명세 기준**: IFMSSTD0001 업종별 투자자별 통계 = 105바이트
- **포함 정보**: 업종코드, 매수수량, 매도수량, 거래대금 등 (정확한 오프셋은 KOSCOM 인터페이스정의서 참조)

### 3.3 23104 로그에는 외국인 데이터 없음

23104는 **실시간 호가** 전용 채널이다.

| TR코드 | 건수 | 설명 |
|--------|------|------|
| B601Q | 700만+ | 코스닥 우선호가 |
| B801Q | 5만+ | 코스닥 장개시전 단일가 호가잔량 |
| B601X | 9천+ | 코넥스 우선호가 |
| LK000 | 340 | Polling |

→ 23104는 **호가/잔량**만 포함하며, 투자자 구분 정보가 없다.

## 4. 수집 방법

### 4.1 수집 대상 채널

| 목적 | 포트 | TR코드 | 비고 |
|------|------|--------|------|
| **업종별** 외국인 수급 | **23103** | C001Q102 | 코스닥 |
| 업종별 외국인 수급 | 22103 | C001S102 | 코스피 |
| 업종별 외국인 수급 | 23103 | C001X102 | 코넥스 |

### 4.2 파싱 절차

```
1. 로그 파일 열기 (바이너리)
2. 0xFF로 스플릿 → 레코드 리스트
3. 각 레코드:
   - r[:6] == b'C001Q1' (또는 C001S1, C001X1) 확인
   - r[6:9] == b'02' → 외국인 (102)
   - r[9:] → 업종코드, 매수/매도 수량, 금액 등 파싱
4. 타임스탬프 추출 (레코드 마지막 13자리)
```

### 4.3 파싱 예시 (Python)

```python
def parse_foreign_from_log(log_path: str):
    with open(log_path, "rb") as f:
        data = f.read()
    records = data.split(b'\xff')
    foreign = []
    for r in records:
        if not r.startswith(b'C001Q102'):  # 코스닥 외국인
            continue
        # r[9:] 파싱 (업종코드, 매수수량, 매도수량 등)
        # 정확한 오프셋은 KOSCOM 인터페이스정의서 확인 필요
        ts_ms = int(r[-13:].decode()) if len(r) >= 13 else None
        foreign.append({"ts_ms": ts_ms, "raw": r[9:-13].decode()})
    return foreign
```

### 4.4 C001 한계: 업종 단위

- C001은 **업종별** 집계이므로 **종목 단위** 외국인 수급은 아니다.
- 종목별 데이터가 필요하면 **C101**이 수신되는 채널을 사용해야 한다.

## 5. fam/foreign 폴더와의 관계

### 5.1 현재 foreign_flow.parquet 구조

- **종목 단위** (symbol = ISIN)
- **실시간/틱** 수준 (종목당 평균 1,135건/일)
- 컬럼: foreign_buy, foreign_sell, foreign_net, foreign_buy_qty, foreign_sell_qty, foreign_net_qty 등

### 5.2 데이터 소스 추정

- C001은 **업종별**이므로, `foreign_flow.parquet`의 **종목별** 데이터는 C001과 직접 대응되지 않는다.
- 가능한 소스:
  1. **C101** (종목별 투자자별 종가통계) — 배치 채널
  2. **실시간 체결 + 투자자 구분** — 별도 전문 (예: A301 + 투자자 플래그)
  3. **외부 API/데이터** — KRX 데이터마켓플레이스 등

### 5.3 C001로 할 수 있는 것

- 업종별 외국인 순매수/순매도 시계열
- 90초 주기 업종 수급 모니터링
- **종목별** 데이터는 C001만으로는 불가

## 8. 밀리초 단위 외국인 수급 가능 여부

### 8.1 표준 KOSCOM 로그 기준: **불가**

| 데이터 소스 | 시간 해상도 | 투자자 구분 | 밀리초 수급 |
|-------------|-------------|-------------|-------------|
| **C001** (업종별 투자자별) | **90초** 주기 | ✅ 있음 | ❌ 불가 |
| **A301** (체결) | 실시간 (ms) | ❌ **없음** | ❌ 불가 |
| C101 (종목별) | 15:30, 18:00 배치 | ✅ 있음 | ❌ 불가 |

- **A301 체결** 레코드(186바이트)에는 **투자자구분코드가 포함되지 않는다**.
- 체결 시점(ms)은 알 수 있으나, 해당 체결이 외국인/기관/개인 중 누구인지는 표준 전문에 없다.
- 따라서 **표준 로그만으로는 밀리초 단위 외국인 수급을 복원할 수 없다**.

### 8.2 fam/foreign 데이터: **서브초 가능**

`fam/foreign/foreign_flow.parquet` 분석 결과:

| 항목 | 값 |
|------|-----|
| 최소 레코드 간격 | **638ms** |
| 중앙값 간격 | ~10.6초 |
| 2초 이내 간격 비율 | ~8% |

→ 이 데이터는 **서브초(638ms~) 수준** 갱신을 지원한다.  
→ 단, **수집 소스는 현재 로그와 다르다** (C001 90초, A301 무투자자구분과 불일치).

### 8.3 밀리초 수급이 가능한 경로 (추정)

| 경로 | 설명 |
|------|------|
| **체결+투자자 통합 전문** | 체결마다 투자자 구분을 포함하는 비표준/전용 상품 (명세 미확인) |
| **증권사/벤더 전용 피드** | 일부 증권사·데이터 벤더가 제공하는 실시간 투자자별 체결 |
| **KOSCOM 오픈API** | 유가종목별투자자 등 API — 장마감 후 제공, 실시간 아님 |

### 8.4 결론

- **표준 KOSCOM 로그(20250102 3)**: 밀리초 단위 외국인 수급 **불가**.
- **fam/foreign**: 서브초 수준 갱신 가능하나, **해당 로그와는 다른 파이프라인**에서 생성된 것으로 추정.
